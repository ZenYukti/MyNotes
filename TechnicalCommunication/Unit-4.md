## 1. Leadership Communication

### 1.1 What is Leadership Communication?

**Definition**: Strategic use of communication to inspire, influence, and guide teams toward shared goals

```
      Leadership Communication
                │
    ┌───────────┼───────────┬────────────┬─────────────┐
    │           │           │            │             │
  Vision     Inspire    Influence   Feedback    Crisis
Articulation  Teams                             Management
```

**Difference: Leader vs Manager Communication**

| Aspect | Manager Communication | Leader Communication |
|--------|----------------------|---------------------|
| **Focus** | Tasks, processes | Vision, inspiration |
| **Direction** | Top-down orders | Collaborative dialogue |
| **Purpose** | Control, compliance | Empower, motivate |
| **Style** | Transactional | Transformational |
| **Questions** | "How & when?" | "Why & what if?" |

```

│                                                        │

**Example - Transformational Communication:**

```
❌ Transactional (Manager):
"Team, we need to increase sales by 20% this quarter. 
Work harder. That's the target."

✓ Transformational (Leader):
"Team, I want to share something exciting. [VISION]

We have an opportunity to help 10,000 more customers 
transform their businesses with our solution. That's 
10,000 success stories—families fed, dreams realized. 
[PURPOSE]

To make this happen, we need to grow sales by 20% this 
quarter. I know it's ambitious, but I've seen what you're 
capable of. Remember last quarter when we thought 15% was 
impossible? You crushed it. [CHALLENGE + BELIEF]

I'm here to support you. Let's meet weekly to remove 
obstacles. Marketing will provide new materials. We're 
investing in training. [SUPPORT]

Let's make this quarter legendary. Who's with me?"
[INSPIRATION]

✓ Emotion + Purpose
✓ Belief in team
✓ Support offered
✓ Call to action
```

```

**Example - Crisis Communication:**

```
Scenario: Company layoffs due to financial crisis

❌ Poor Crisis Communication:
"Due to budget issues, we're letting 100 people go. 
Check your email to see if you're affected. Thanks."

✓ Effective Crisis Communication:

"Team, I need to share difficult news. [ACKNOWLEDGE]

Our Q4 revenue fell 40% short of projections. Despite 
cost-cutting efforts over 3 months, we must reduce our 
workforce by 100 positions to ensure company survival. 
[INFORM - TRANSPARENT]

I know this news is devastating. Many of you have friends 
affected. This is heartbreaking for all of us, especially 
those losing jobs. [EMPATHIZE]

Here's what happens next:
• Today, 3 PM: Affected employees notified personally
• This week: Severance packages (3 months + benefits)
• Next month: Career counseling, job placement support
[DIRECTION]

For those staying, I'm committed to rebuilding trust. 
I'll hold weekly Q&A sessions. No question off-limits. 
[TRANSPARENCY]

We will emerge stronger. I believe in this team and our 
ability to recover. [REASSURE]

I'll update you Friday with recovery plan details. My 
door is always open. [FOLLOW-UP]"

✓ Honest and direct
✓ Empathetic tone
✓ Clear action plan
✓ Commitment to transparency
```

## 2. Communication and Social Competence

### 2.1 What is Social Competence?

**Definition**: Ability to effectively navigate social situations, understand others, and build positive relationships

```
        Social Competence
              │
    ┌─────────┼─────────┬──────────┬───────────┐
    │         │         │          │           │
  Context   Feelings  Intentions Behaviors   Outcomes
 Awareness Recognition  Reading    Adapt     Positive
                                              Relations
```

### 2.2 The Four Pillars of Social Competence

```
┌────────────────────────────────────────────────────────┐
│                                                        │
│  1. CONTEXT (Situational Awareness)                    │
│     ├─ Where am I?                                    │
│     ├─ Who is present?                                │
│     ├─ What are social norms here?                    │
│     └─ What's appropriate?                            │
│                                                        │
│  Example:                                              │
│  Job interview → Formal language, professional tone    │
│  Friend's party → Casual, relaxed communication        │
│                                                        │
├────────────────────────────────────────────────────────┤
│                                                        │
│  2. FEELINGS (Emotional Intelligence)                  │
│     ├─ What am I feeling?                             │
│     ├─ What are others feeling?                       │
│     ├─ How do emotions affect communication?          │
│     └─ Can I manage emotions appropriately?           │
│                                                        │
│  Example:                                              │
│  Colleague seems upset → Approach with empathy:        │
│  "You seem stressed. Want to talk?"                    │
│                                                        │
├────────────────────────────────────────────────────────┤
│                                                        │
│  3. INTENTIONS (Understanding Motives)                 │
│     ├─ What do I want to achieve?                     │
│     ├─ What are others' goals?                        │
│     ├─ Are there hidden agendas?                      │
│     └─ How can I align interests?                     │
│                                                        │
│  Example:                                              │
│  Boss criticizes work → Intention: Help improve,       │
│  not attack personally. Respond constructively.        │
│                                                        │
├────────────────────────────────────────────────────────┤
│                                                        │
│  4. BEHAVIORS (Action & Adaptation)                    │
│     ├─ What behaviors are appropriate?                │
│     ├─ How should I respond?                          │
│     ├─ Can I adapt my style?                          │
│     └─ What are the consequences?                     │
│                                                        │
│  Example:                                              │
│  Presenting to executives → Adjust: Data-driven,       │
│  concise, focus on ROI (not technical jargon)          │
│                                                        │
└────────────────────────────────────────────────────────┘
```

### 2.4 Recognizing and Managing Feelings

**Emotional Intelligence in Communication:**

```
        Emotional Intelligence
                │
    ┌───────────┼───────────┬──────────┬───────────┐
    │           │           │          │           │
Self-       Self-      Social      Relationship
Awareness  Regulation  Awareness   Management
    │           │           │          │
  "I feel"  "I control" "You feel" "We resolve"
```

**Examples:**

**A. Self-Awareness:**
```
Low Self-Awareness:
"I don't know why I snapped at my team. They're just 
annoying today."

High Self-Awareness:
"I notice I'm irritable because I didn't sleep well and 
I'm stressed about the deadline. I need to manage this 
before it affects my team interactions."
```

**B. Self-Regulation:**
```
❌ Poor Regulation:
Team member makes mistake.
You: "How could you be so careless? This is unacceptable!"
(Anger expressed inappropriately)

✓ Good Regulation:
You feel angry, but pause. Take a breath.
You: "I see an error here. Let's discuss what happened 
     and how to prevent it. Walk me through your process."
(Constructive, calm, solution-focused)
```

**C. Social Awareness (Empathy):**
```
Low Empathy:
Colleague: "I'm struggling with this task."
You: "Just figure it out. It's not that hard."

High Empathy:
Colleague: "I'm struggling with this task."
You: "I can see you're frustrated. What specific part 
     is challenging? Let me help."
```

**D. Relationship Management:**
```
Conflict Scenario: Disagreement with coworker

Poor Management:
"You're wrong. My way is better. Just do it."
(Damages relationship)

Effective Management:
"I understand your perspective. Here's my concern: [explain]. 
Can we find a solution that addresses both our needs? 
What if we tried [compromise]?"
(Maintains relationship, collaborative)
```

### 2.5 Reading Intentions

**Understanding Underlying Motives:**

```
Surface Message vs Underlying Intention

Scenario 1:
Message: "Your presentation was... interesting."
(Said with hesitation)

Reading Intention:
Likely Intention: Didn't like it, being polite
Response: "I sense some reservations. I'd appreciate 
          honest feedback to improve."

──────────────────────────────────────────────────────

Scenario 2:
Boss: "Can you take on this urgent project?"
(Friday, 5 PM)

Reading Intention:
Possible Intentions:
• Crisis genuinely requires your expertise
• Testing commitment/work ethic
• Poor planning on their part

Response: "I want to help. Can you share more context? 
          What's the deadline and priority? I have 
          X commitment this weekend—can we discuss 
          timeline?"
(Clarify intention, set boundaries)

──────────────────────────────────────────────────────

Scenario 3:
Colleague: "I don't need your help."
(Arms crossed, defensive tone)

Reading Intention:
Likely: Feels threatened, insecure, or proud
Not: Genuinely doesn't need help

Response: "I understand. Just want you to know I'm here 
          if anything comes up. You've got this."
(Respect boundary, leave door open)
```

### 2.6 Adapting Behaviors

**Communication Flexibility:**

```
Situation → Behavior Adaptation

AUDIENCE: Technical Engineers
Adapt To:
├─ Use technical jargon appropriately
├─ Provide detailed specifications
├─ Back claims with data
└─ Encourage technical debate

Example: "The API latency is 45ms at p95. We optimized 
         using connection pooling and caching. Thoughts?"

──────────────────────────────────────────────────────

AUDIENCE: Non-Technical Executives
Adapt To:
├─ Simplify technical concepts
├─ Focus on business impact (ROI, time)
├─ Use analogies
└─ Bottom-line first

Example: "Our system improvements save $200K annually 
         by reducing server costs 40%. Users experience 
         2x faster load times."

──────────────────────────────────────────────────────

SITUATION: Team is Demotivated
Adapt To:
├─ Energetic, positive tone
├─ Acknowledge challenges empathetically
├─ Celebrate small wins
└─ Inspire with vision

Example: "I know this week was tough. But look—we fixed 
         50 bugs! That's dedication. Imagine how our users 
         will feel with this stable product. Let's keep 
         the momentum!"

──────────────────────────────────────────────────────

SITUATION: Crisis/High Pressure
Adapt To:
├─ Calm, measured tone
├─ Clear, directive communication
├─ Focus on solutions
└─ Provide reassurance

Example: "Here's the situation. Server is down. John, 
         restore backup. Sarah, call clients. I'll handle 
         the CEO. We'll resolve this in 2 hours. You've 
         got this."
```

## 3. Providing and Receiving Feedback

### 3.1 Importance of Feedback

```
        Feedback Purpose
              │
    ┌─────────┼─────────┬──────────┬───────────┐
    │         │         │          │           │
  Improve   Reinforce  Build     Align       Develop
Performance Strengths  Trust   Expectations  Growth
```

### 3.2 Providing Feedback - The SBI Model

**SBI Framework: Situation-Behavior-Impact**

```
┌────────────────────────────────────────────┐
│ SITUATION                                  │
│ When/Where did it happen?                  │
│ Provide context, be specific              │
└────────┬───────────────────────────────────┘
         │
         ▼
┌────────────────────────────────────────────┐
│ BEHAVIOR                                   │
│ What exactly did the person do/say?        │
│ Observable actions, not judgments          │
└────────┬───────────────────────────────────┘
         │
         ▼
┌────────────────────────────────────────────┐
│ IMPACT                                     │
│ What was the effect/consequence?           │
│ How did it affect you, team, project?      │
└────────────────────────────────────────────┘
```

**Examples:**

**Positive Feedback:**
```
❌ Vague:
"Great job on the project!"

✓ Using SBI:
SITUATION: "In yesterday's client meeting,"
BEHAVIOR: "you proactively addressed their concerns about 
           data security and provided a detailed solution,"
IMPACT: "which built their confidence. As a result, they 
         signed the contract. Your preparation made the 
         difference."

✓ Specific
✓ Highlights exact behavior
✓ Shows impact
```

**Constructive Feedback:**
```
❌ Judgmental:
"You're always late and irresponsible."

✓ Using SBI:
SITUATION: "Over the past two weeks, in our 9 AM standups,"
BEHAVIOR: "you've arrived 15-20 minutes late on 6 occasions,"
IMPACT: "which delays the meeting for the entire team of 10 
         people. We lose productivity, and it affects morale. 
         Can we discuss what's preventing you from arriving 
         on time?"

✓ Non-judgmental
✓ Specific instances
✓ Clear impact
✓ Invites dialogue
```

### 3.3 Feedback Sandwich (Use Cautiously)

```
         Feedback Sandwich
              │
    ┌─────────┼─────────┐
    │         │         │
Positive   Negative   Positive
 (Praise)  (Criticism) (Hope)
```

**Example:**
```
"Raj, I really appreciate your creativity in design. Your 
UI mockups are always visually stunning. [POSITIVE]

However, I've noticed the last three designs weren't 
responsive on mobile devices, which is critical for our 
users. This needs improvement. [CONSTRUCTIVE]

I know you have the skills to master responsive design. 
With some training, you'll create exceptional work on 
all platforms. [POSITIVE + FORWARD]"

⚠ Warning: Overuse diminishes impact. People start 
           expecting bad news after praise. Use 
           authentically, not formulaically.
```

### 3.4 Receiving Feedback Gracefully

```
Steps to Receive Feedback Well:
┌────────────────────────────────────────┐
│ 1. LISTEN ACTIVELY                     │
│    Don't interrupt, nod, make eye contact│
└────────┬───────────────────────────────┘
         │
         ▼
┌────────────────────────────────────────┐
│ 2. DON'T GET DEFENSIVE                 │
│    Pause, breathe, resist justifying   │
└────────┬───────────────────────────────┘
         │
         ▼
┌────────────────────────────────────────┐
│ 3. CLARIFY                             │
│    Ask questions to understand fully   │
└────────┬───────────────────────────────┘
         │
         ▼
┌────────────────────────────────────────┐
│ 4. ACKNOWLEDGE                         │
│    Thank the person for feedback       │
└────────┬───────────────────────────────┘
         │
         ▼
┌────────────────────────────────────────┐
│ 5. REFLECT                             │
│    Consider validity, identify truth   │
└────────┬───────────────────────────────┘
         │
         ▼
┌────────────────────────────────────────┐
│ 6. ACT                                 │
│    Create plan, implement changes      │
└────────────────────────────────────────┘
```

**Example Dialogue:**

```
Boss: "Your reports often come in late, and they lack 
       detail in the analysis section."

❌ Defensive Response:
"That's not true! I submit on time. Besides, I'm too busy 
with other tasks. No one told me about detail requirements."

✓ Graceful Response:

[LISTEN] (Nod, maintain eye contact)

[DON'T DEFEND] (Pause, breathe)
"I appreciate you bringing this up."

[CLARIFY]
"Can you help me understand? Which specific reports were 
late, and what level of detail are you expecting in analysis?"

Boss: "Last 3 monthly reports were 2-3 days late. For analysis, 
       I need root cause examination, not just data presentation."

[ACKNOWLEDGE]
"Thank you for the specific examples. I understand now."

[REFLECT]
"You're right—I did miss those deadlines. I can improve 
analysis depth."

[ACT]
"Here's my plan: I'll set reminders 5 days before deadlines 
and review past reports to match analysis depth. Can I send 
you next month's report early for feedback?"

✓ Professional
✓ Non-defensive
✓ Action-oriented
✓ Growth mindset
```

## 4. Difference Between Tact and Intelligence

### 4.1 Definitions

```
┌────────────────────────────────────────────────────┐
│ TACT                                               │
│ Social skill of saying the right thing at the     │
│ right time without offending                       │
│                                                    │
│ Key Elements:                                      │
│ • Sensitivity to others' feelings                 │
│ • Diplomatic language                             │
│ • Situational awareness                           │
│ • Empathy in communication                        │
└────────────────────────────────────────────────────┘

┌────────────────────────────────────────────────────┐
│ INTELLIGENCE                                       │
│ Cognitive ability to learn, reason, and solve     │
│ problems                                           │
│                                                    │
│ Key Elements:                                      │
│ • Logical reasoning                               │
│ • Problem-solving                                 │
│ • Knowledge acquisition                           │
│ • Analytical thinking                             │
└────────────────────────────────────────────────────┘
```

### 4.2 Comparison

| Aspect | Tact | Intelligence |
|--------|------|--------------|
| **Nature** | Social/Emotional skill | Cognitive ability |
| **Focus** | How you say it | What you know |
| **Goal** | Maintain harmony | Solve problems |
| **Measurement** | Social success | IQ tests, grades |
| **Development** | Practice, experience | Education, training |
| **Application** | Interpersonal situations | Technical/academic tasks |

### 4.3 Examples

**Scenario 1: Colleague's Idea is Flawed**

```
LOW TACT, HIGH INTELLIGENCE:
"That idea is stupid. It won't work because of X, Y, Z. 
Anyone with basic knowledge would know this."

✓ Correct analysis (intelligence)
❌ Offensive delivery (no tact)
Result: Relationship damaged, colleague defensive

──────────────────────────────────────────────────────

HIGH TACT, LOW INTELLIGENCE:
"That's an interesting idea! I'm not sure about the 
technical details, but it sounds promising!"

✓ Polite, encouraging (tact)
❌ Doesn't identify flaw (limited intelligence)
Result: Flawed idea proceeds, causes problems later

──────────────────────────────────────────────────────

HIGH TACT, HIGH INTELLIGENCE:
"I appreciate your creative thinking! I see the potential 
here. I have some concerns about implementation—specifically 
X, Y, and Z might cause issues. What if we adjusted the 
approach to address these? I think we can make this work."

✓ Polite, validating (tact)
✓ Identifies problems (intelligence)
✓ Offers solution (both)
Result: Maintains relationship + Improves idea
```

**Scenario 2: Employee Makes Repeated Mistakes**

```
HIGH INTELLIGENCE, LOW TACT:
"You keep making the same errors. Are you even trying? 
This is basic stuff."

Outcome: Employee feels attacked, motivation drops

──────────────────────────────────────────────────────

LOW INTELLIGENCE, HIGH TACT:
"It's okay! Everyone makes mistakes. Don't worry about it!"

Outcome: Problem not addressed, mistakes continue

──────────────────────────────────────────────────────

HIGH INTELLIGENCE, HIGH TACT:
"I've noticed a pattern in these errors. Let's work through 
this together. Can you walk me through your process? I think 
we can identify where things are going wrong and create a 
checklist to prevent it. Sound good?"

Outcome: Problem addressed constructively, employee supported
```

### 4.4 Why Both Matter

```
Intelligence Without Tact = Smart but Alienating
  "Brilliant jerk" syndrome
  Correct but offensive
  Knowledge without wisdom

Tact Without Intelligence = Nice but Ineffective
  Pleasant but doesn't solve problems
  Avoids necessary tough conversations
  Style over substance

Intelligence + Tact = Effective Leadership
  Solve problems diplomatically
  Inspire and influence
  Build trust while driving results
```

### 5.2 Trust Through Communication

**Building Trust:**

```
        Trust = Credibility + Reliability + Intimacy
                    ──────────────────────────────
                          Self-Orientation

Credibility: Competence, expertise
Reliability: Consistency, follow-through
Intimacy: Open, authentic communication
Self-Orientation: Focus on others, not just yourself
```

**Trust-Building Communication Practices:**

```
1. TRANSPARENCY
   ├─ Share information openly
   ├─ Admit mistakes
   ├─ Explain reasoning
   └─ No hidden agendas

   Example: "I made an error in the budget forecast. Here's 
            what happened and how I'm fixing it."

2. CONSISTENCY
   ├─ Match words with actions
   ├─ Deliver on promises
   ├─ Maintain same values
   └─ Predictable responses

   Example: Say "I'll respond by tomorrow" → Do it

3. VULNERABILITY
   ├─ Share challenges
   ├─ Ask for help
   ├─ Admit you don't know
   └─ Show authentic self

   Example: "I'm struggling with this problem. Can anyone 
            help me think through it?"

4. ACTIVE LISTENING
   ├─ Give full attention
   ├─ Validate feelings
   ├─ Don't interrupt
   └─ Remember details

   Example: "So what I'm hearing is you're concerned about 
            X. Is that right? Tell me more."

5. FOLLOW-THROUGH
   ├─ Keep commitments
   ├─ Meet deadlines
   ├─ Update on progress
   └─ Apologize if delayed

   Example: "I committed to the report by Friday. It's ready."
```

**Trust-Destroying Behaviors:**

```
❌ Gossip
❌ Broken promises
❌ Taking credit for others' work
❌ Inconsistency (say one thing, do another)
❌ Hiding information
❌ Blaming others
❌ Not listening
```

## 6. Thinking Skills

### 6.1 What are Thinking Skills?

**Definition**: Cognitive abilities used to process information, solve problems, and make decisions

```
        Thinking Skills
              │
    ┌─────────┼─────────┬──────────┬───────────┐
    │         │         │          │           │
 Critical  Creative  Analytical  Reflective
Thinking   Thinking   Thinking    Thinking
```

### 6.2 Types of Thinking

#### A. Critical Thinking

**Definition**: Objective analysis and evaluation of an issue to form judgment

```
Critical Thinking Process:
┌────────────────────────────────────┐
│ 1. Identify Issue/Claim            │
└────────┬───────────────────────────┘
         │
         ▼
┌────────────────────────────────────┐
│ 2. Gather Information               │
└────────┬───────────────────────────┘
         │
         ▼
┌────────────────────────────────────┐
│ 3. Evaluate Evidence                │
│    • Is it reliable?                │
│    • Is it biased?                  │
│    • Is it relevant?                │
└────────┬───────────────────────────┘
         │
         ▼
┌────────────────────────────────────┐
│ 4. Consider Alternatives            │
└────────┬───────────────────────────┘
         │
         ▼
┌────────────────────────────────────┐
│ 5. Draw Conclusion                  │
└────────────────────────────────────┘
```

**Example:**

```
Claim: "AI will replace all programmers by 2030."

Critical Thinking Analysis:
├─ Evidence: What data supports this?
├─ Source Credibility: Who made this claim? Expert? Clickbait?
├─ Logic: Is the reasoning sound?
├─ Assumptions: What's assumed? (AI capability, timeline)
├─ Counterevidence: What contradicts this?
├─ Alternatives: Maybe AI augments, not replaces?

Conclusion: "AI will automate routine coding tasks, but 
complex problem-solving, architecture, and creativity will 
still require human programmers. Full replacement by 2030 
is unlikely."
```

#### B. Creative Thinking

**Definition**: Generating novel, innovative ideas and solutions

```
Creative Thinking Techniques:

1. BRAINSTORMING
   ├─ Generate many ideas
   ├─ No judgment initially
   ├─ Build on others' ideas
   └─ Encourage wild ideas

2. LATERAL THINKING
   ├─ Challenge assumptions
   ├─ Explore unconventional paths
   ├─ Connect unrelated concepts
   └─ Ask "What if?"

3. MIND MAPPING
   ├─ Central idea in center
   ├─ Branch out with associations
   ├─ Visual connections
   └─ Nonlinear exploration

4. SCAMPER (Improvement technique)
   ├─ Substitute
   ├─ Combine
   ├─ Adapt
   ├─ Modify
   ├─ Put to other use
   ├─ Eliminate
   └─ Reverse
```

**Example:**

```
Problem: Low employee engagement

Creative Solutions (Brainstorming):
├─ Gamify work tasks (points, leaderboards)
├─ Reverse mentoring (junior teach senior)
├─ "Failure Fridays" (celebrate lessons from failures)
├─ Job rotation program
├─ Anonymous CEO Q&A via app
├─ Employee-led innovation time (20% projects)
├─ Wellness challenges with team competition
├─ Pet-friendly office days
└─ Learning budget ($1000/year per employee)
```

#### C. Analytical Thinking

**Definition**: Breaking down complex information into smaller parts to understand structure and relationships

```
Analytical Process:
┌────────────────────────────────┐
│ 1. Break Down Problem          │
│    (Decompose into components) │
└────────┬───────────────────────┘
         │
         ▼
┌────────────────────────────────┐
│ 2. Examine Each Part           │
│    (Identify patterns, data)   │
└────────┬───────────────────────┘
         │
         ▼
┌────────────────────────────────┐
│ 3. Identify Relationships      │
│    (How parts connect)         │
└────────┬───────────────────────┘
         │
         ▼
┌────────────────────────────────┐
│ 4. Draw Insights               │
│    (Synthesize understanding)  │
└────────────────────────────────┘
```

**Example:**

```
Problem: Website conversion rate dropped from 5% to 2%

Analytical Breakdown:
├─ Traffic source analysis
│  ├─ Organic: Same volume, lower conversion
│  ├─ Paid ads: Higher volume, much lower conversion
│  └─ Social: Unchanged
│
├─ User behavior analysis
│  ├─ Bounce rate increased on landing page
│  ├─ Average time on site decreased
│  └─ Cart abandonment rate up 40%
│
├─ Technical analysis
│  ├─ Page load time increased to 8 seconds
│  ├─ Mobile responsiveness issues
│  └─ Payment gateway errors
│
└─ Insight: Recent site redesign slowed load time, 
    especially on mobile (70% of paid ad traffic). 
    Fix: Optimize images, enable caching.
```

#### D. Reflective Thinking

**Definition**: Thinking about your own thought processes and experiences to learn and improve

```
Reflective Cycle (Gibbs):
┌────────────────────────────────┐
│ 1. Description                 │
│    What happened?              │
└────────┬───────────────────────┘
         │
         ▼
┌────────────────────────────────┐
│ 2. Feelings                    │
│    What were you thinking?     │
└────────┬───────────────────────┘
         │
         ▼
┌────────────────────────────────┐
│ 3. Evaluation                  │
│    What was good/bad?          │
└────────┬───────────────────────┘
         │
         ▼
┌────────────────────────────────┐
│ 4. Analysis                    │
│    What sense can you make?    │
└────────┬───────────────────────┘
         │
         ▼
┌────────────────────────────────┐
│ 5. Conclusion                  │
│    What else could you do?     │
└────────┬───────────────────────┘
         │
         ▼
┌────────────────────────────────┐
│ 6. Action Plan                 │
│    What will you do next time? │
└────────────────────────────────┘
```

**Example:**

```
Situation: Led team meeting that went poorly

Reflection:
1. DESCRIPTION
   Team meeting lasted 90 min (planned 30). People seemed 
   disengaged. Key decisions weren't made.

2. FEELINGS
   Frustrated, embarrassed. Felt I lost control.

3. EVALUATION
   Bad: No agenda, let discussions drift, didn't manage time
   Good: Everyone participated (at least they talked)

4. ANALYSIS
   Root cause: Lack of structure. I wanted open discussion 
   but no framework led to chaos. Also didn't assign roles 
   (timekeeper, note-taker).

5. CONCLUSION
   Effective meetings need structure + facilitation. 
   Open discussion works within clear boundaries.

6. ACTION PLAN
   Next meeting:
   ├─ Create detailed agenda (share 24h advance)
   ├─ Assign roles (facilitator, timekeeper, scribe)
   ├─ Use parking lot for off-topic items
   ├─ Time-box each topic
   └─ End with clear action items and owners
```

## 🚀 Quick Reference Summary

### Leadership Communication
- **Transformational:** Inspire vision, connect to purpose, support growth
- **Situational:** Adapt style to team maturity (Directing → Coaching → Supporting → Delegating)
- **Crisis:** Acknowledge → Inform → Empathize → Direct → Reassure → Follow-up

### Social Competence (4 Pillars)
1. **Context:** Read the room, understand norms
2. **Feelings:** Recognize and manage emotions
3. **Intentions:** Understand underlying motives
4. **Behaviors:** Adapt communication style appropriately

### Feedback
**Giving (SBI):** Situation → Behavior → Impact
**Receiving:** Listen → Don't defend → Clarify → Acknowledge → Reflect → Act

### Tact vs Intelligence
- **Tact:** Social skill, how you say it, maintains harmony
- **Intelligence:** Cognitive ability, what you know, solves problems
- **Best:** Both together for effective leadership

### Emotional Intelligence
- Self-Awareness + Self-Regulation + Social Awareness + Relationship Management
- **Trust formula:** (Credibility + Reliability + Intimacy) / Self-Orientation

### Thinking Skills
- **Critical:** Analyze, evaluate evidence, form judgment
- **Creative:** Generate novel ideas, challenge assumptions
- **Analytical:** Break down, examine parts, identify relationships
- **Reflective:** Learn from experience, improve processes
