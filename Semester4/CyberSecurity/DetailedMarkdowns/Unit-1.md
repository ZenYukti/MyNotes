## 1. Cybercrime — Definition & Origins

### Definition
- **Cybercrime** = any criminal activity that involves a **computer, network, or internet** as a tool, target, or place of crime
- Also called **computer crime** or **e-crime**
- Combines traditional crime with digital technology

### Origin of the Word
- Derived from **"Cyber"** (from Cybernetics — science of communication and control in machines/humans) + **"Crime"**
- Term gained popularity in **1990s** with the rise of the internet
- First known computer crime: **1820** (Joseph-Marie Jacquard's loom sabotage by workers fearing job loss — arguably first)
- Modern cybercrime grew with **email, e-commerce, and social media**

### Information Security
- **Information Security (InfoSec)** = protecting information from unauthorized access, use, disclosure, disruption, modification, or destruction
- **CIA Triad** = core principles of InfoSec:
  - **Confidentiality** — data accessible only to authorized users
  - **Integrity** — data is accurate and unaltered
  - **Availability** — data accessible when needed

## 2. Who are Cybercriminals?

| Type | Description |
|------|-------------|
| **Hackers** | Break into systems (white hat = ethical, black hat = malicious, grey hat = both) |
| **Crackers** | Break security systems with malicious intent |
| **Script Kiddies** | Unskilled; use pre-written hacking tools/scripts |
| **Insiders** | Employees/ex-employees misusing access |
| **Phishers** | Trick users into revealing personal info |
| **Cyberstalkers** | Harass individuals online |
| **State-sponsored hackers** | Government-backed attackers targeting nations |
| **Hacktivists** | Hack for political/ideological reasons (e.g., Anonymous) |
| **Organized Crime Groups** | Professional criminal networks conducting fraud, theft |

## 3. Classifications of Cybercrimes

### Based on Target

| Category | Examples |
|----------|---------|
| **Crime against Individuals** | Cyberstalking, identity theft, harassment, phishing |
| **Crime against Property** | Hacking, virus attacks, IPR violations, online fraud |
| **Crime against Organizations** | Data theft, sabotage, DoS attacks, corporate espionage |
| **Crime against Society/Nation** | Cyber terrorism, spreading hate content, child pornography |

### Based on Type

- **Hacking** — unauthorized access to systems
- **Phishing** — fraudulent emails/websites to steal credentials
- **Identity Theft** — stealing personal info to impersonate
- **Ransomware** — encrypt files, demand ransom
- **DoS/DDoS** — flood server to make it unavailable
- **Cyberbullying** — harassment via digital platforms
- **Online Fraud** — financial scams, fake e-commerce
- **Child Exploitation** — child pornography, grooming
- **Cyber Espionage** — state-level spying via digital means

## 4. A Global Perspective on Cybercrimes

- Cybercrime is a **borderless global threat** — attacker and victim can be in different countries
- **Challenges:**
  - Lack of uniform laws across countries
  - Jurisdiction issues — which country prosecutes?
  - Anonymity of attackers (VPNs, dark web, proxy servers)
  - Rapid technological change outpaces law enforcement
- **Key Global Frameworks:**
  - **Budapest Convention** (2001) — first international treaty on cybercrime
  - **Interpol Cybercrime Unit** — global cooperation
  - **UN GGE** — UN Group of Governmental Experts on cybersecurity norms
- **Most targeted countries:** USA, UK, India, Germany, China
- **India:** IT Act 2000 (amended 2008) governs cybercrime

## 5. Cybercrime Era: Survival Mantra for Netizens

Tips every internet user must follow:

- 🔒 Use **strong passwords** (mix of letters, numbers, symbols); use password manager
- 🔄 **Update software** regularly — patch vulnerabilities
- 🚫 **Don't click unknown links** — phishing prevention
- 🛡️ Use **antivirus and firewall**
- 📧 Be careful with **email attachments** — verify sender
- 🌐 Use **HTTPS** sites only for transactions
- 📱 Enable **two-factor authentication (2FA)**

- 💾 Take regular **data backups**
- 🧠 **Cyber awareness** — stay informed about new threats
- 🚷 Avoid using **public Wi-Fi** for sensitive activities
- 📜 Know your **legal rights** under cyber laws

## 6. How Criminals Plan Attacks

Cybercriminals follow a structured approach:

### Attack Phases (Cyber Kill Chain)
1. **Reconnaissance** — gather information about target (passive: OSINT, active: scanning)
2. **Scanning** — probe target for vulnerabilities (port scanning, vulnerability scanning)
3. **Gaining Access** — exploit vulnerabilities to enter system
4. **Maintaining Access** — install backdoors, rootkits to stay inside
5. **Covering Tracks** — delete logs, hide evidence of intrusion

### Common Reconnaissance Techniques
- **Google Dorking** — advanced Google searches to find vulnerabilities
- **WHOIS lookup** — find domain ownership info
- **Social media profiling** — gather personal info
- **Phishing emails** — trick users into revealing info
- **Port scanning** — tools like Nmap to find open ports

## 7. Social Engineering

### Definition
- **Social Engineering** = manipulating **people** (not machines) into revealing confidential information or performing actions
- Exploits **human psychology** — trust, fear, urgency, authority
- "Hacking the human mind"

### Common Social Engineering Attacks

| Attack | Description |
|--------|-------------|
| **Phishing** | Fake emails pretending to be trusted source (bank, PayPal) |
| **Spear Phishing** | Targeted phishing at specific individual/organization |
| **Vishing** | Voice phishing — phone calls pretending to be bank/IT support |
| **Smishing** | SMS phishing — fake text messages with malicious links |
| **Pretexting** | Creating fake scenario to extract info ("I'm from IT, need your password") |
| **Baiting** | Leave infected USB drive in public — curious user plugs it in |
| **Tailgating/Piggybacking** | Physically follow authorized person into restricted area |
| **Quid Pro Quo** | Offer service in exchange for info (fake tech support) |

### Why it Works
- People are the **weakest link** in security
- Exploits emotions: curiosity, fear, greed, urgency, helpfulness

### Prevention
- **Awareness training** for employees
- **Verify identity** before giving information
- Never give passwords over phone/email
- Be skeptical of unsolicited requests

## 8. Cyberstalking

### Definition
- **Cyberstalking** = use of internet, email, social media, or other digital means to **harass, threaten, or stalk** an individual
- Can be combined with offline stalking

### Characteristics
- Repeated harassment using digital tools
- Can be anonymous
- Victim often knows the stalker (ex-partner, obsessed person)

### Methods Used
- Sending threatening/harassing emails
- Monitoring victim's social media activity
- Tracking location via GPS/check-ins
- Hacking victim's accounts
- Posting false information about victim online
- Subscribing victim to unwanted services

### Types of Cyberstalkers
- **Intimate partner stalkers** — ex-partners
- **Delusional stalkers** — believe they have a relationship with victim
- **Vengeful stalkers** — seeking revenge
- **Predatory stalkers** — planning assault/attack

### Legal Protection (India)
- **IT Act 2000 Section 67** — obscene electronic content
- **IPC Section 354D** — stalking (added 2013 after Nirbhaya case)

## 9. Cybercafé and Cybercrimes

- **Cybercafés** (internet cafés) = shared public computers connected to internet
- Common hotspots for cybercrime due to:
  - **Anonymity** — hard to trace actual user
  - **Shared computers** — malware, keyloggers installed by previous users
  - **No ownership of activity** — criminals use to commit crimes without traceable identity
  - **Weak security** — cafés rarely have proper security measures

### Crimes common at cybercafés:
- Identity theft (saved passwords, browsing history)
- Sending threatening/harassing messages anonymously
- Downloading/distributing illegal content
- Accessing others' email/social accounts
- Running phishing campaigns

### Prevention Measures for Cybercafés:
- **Log all user sessions** with ID proof
- Install **antivirus and firewalls**
- Enable **content filtering**
- **Clear browsing data** after each session
- **CCTV surveillance**
- Users should: always **log out**, never save passwords, use private browsing

## 10. Botnets — The Fuel for Cybercrime

### Definition
- **Botnet** = a network of **compromised computers (bots/zombies)** controlled remotely by an attacker (botmaster)
- Infected machines unaware they are part of the botnet
- Controlled via **Command and Control (C&C) server**

```
Attacker (Botmaster)
       ↓
  C&C Server
   ↙  ↓  ↘
Bot1 Bot2 Bot3 ... (thousands of infected PCs)
       ↓
   Attack Target
```

### How Devices Get Infected
- Malware via **email attachments**
- Drive-by downloads from malicious websites
- Exploiting unpatched vulnerabilities
- Infected USB drives

### What Botnets Are Used For
| Use | Description |
|-----|-------------|
| **DDoS attacks** | Flood target with traffic to crash it |
| **Spam campaigns** | Send millions of spam/phishing emails |
| **Credential theft** | Steal login info from infected machines |
| **Cryptomining** | Use victim's CPU to mine cryptocurrency |
| **Click fraud** | Fake ad clicks for revenue |
| **Ransomware delivery** | Distribute ransomware to victims |
| **Proxy services** | Route illegal traffic through compromised machines |

### Famous Botnets
- **Mirai** — infected IoT devices, caused massive DDoS (2016)
- **Zeus** — banking trojan botnet
- **Conficker** — infected millions of Windows PCs

### Prevention
- Keep OS and software **updated**
- Use **antivirus/anti-malware**
- Avoid suspicious downloads
- **Network monitoring** for unusual traffic

## 11. Attack Vectors

### Definition
- **Attack Vector** = the **path or method** used by an attacker to gain unauthorized access to a system
- Different from **Attack Surface** (total area exposed to attack)

### Common Attack Vectors

| Attack Vector | Description |
|---------------|-------------|
| **Phishing** | Deceptive emails/sites to steal credentials |
| **Malware** | Viruses, trojans, worms, ransomware |
| **Man-in-the-Middle (MitM)** | Intercept communication between two parties |
| **SQL Injection** | Inject malicious SQL code into database queries |
| **Cross-Site Scripting (XSS)** | Inject malicious scripts into web pages |
| **Brute Force** | Try all password combinations until correct one found |
| **Denial of Service (DoS/DDoS)** | Overwhelm server with traffic |
| **Zero-Day Exploit** | Attack on unknown/unpatched vulnerability |
| **Insider Threat** | Malicious action by trusted employee |
| **Physical Access** | Steal device, plug in malicious USB |
| **Social Engineering** | Manipulate humans to bypass security |
| **DNS Poisoning** | Corrupt DNS to redirect users to fake sites |
| **Session Hijacking** | Steal authenticated session tokens |

### Attack Vector Categories
- **Network-based** — exploiting network protocols (DoS, MitM, DNS poisoning)
- **Software-based** — exploiting code vulnerabilities (SQL injection, XSS, zero-day)
- **Human-based** — exploiting people (phishing, social engineering, insider threats)
- **Physical** — direct hardware access
